## 2. 项目背景

本项目来源于机器人自动化抓取场景中“非结构化、多物体堆叠目标抓取”的实际需求。传统机械臂抓取通常依赖人工示教、固定工装、规则摆放和预设抓取点，适用于单一物体、固定位置、姿态稳定的场景。但在实际生产、仓储分拣、实验室自动化和柔性制造中，目标物体往往存在随机摆放、相互遮挡、堆叠挤压、姿态不确定等情况，传统规则方法难以稳定确定抓取动作和抓取位姿。

项目拟通过 ROS2 仿真环境构建机械臂抓取系统，结合 Gazebo / Isaac Sim / MoveIt 等仿真与运动规划工具，对机械臂、相机、夹爪和目标物体进行虚拟建模与验证。在视觉感知层面，引入 GraspNet 等基于 RGB-D 图像或点云的抓取位姿估计方法，使系统能够通过摄像头获取当前场景信息，识别可抓取区域，生成候选抓取姿态，并结合碰撞检测、运动规划和执行约束，最终映射为机械臂可执行的抓取动作。

本项目的典型业务场景包括：仓储物流中的杂乱物品分拣，工业产线中的无序上料，机械零件的料框抓取，实验室自动化中的样本或耗材抓取，以及服务机器人在复杂环境中的目标拾取。这类场景的共同特点是目标位置不固定、物体姿态不确定、多个物体之间存在遮挡和堆叠，系统不能依赖人工预先设定每一个抓取点，而需要根据实时视觉数据自主判断“抓哪里、怎么抓、是否能抓、抓取后如何避障”。

当前痛点主要体现在四个方面。第一，传统视觉方法通常只能识别物体的位置或轮廓，难以直接输出稳定可靠的六自由度抓取位姿。第二，多物体堆叠情况下存在遮挡、接触和重叠，单纯依靠二维图像难以判断物体的空间关系和可抓取性。第三，抓取动作不仅需要视觉识别，还必须考虑机械臂运动范围、夹爪开合约束、碰撞风险和抓取成功率。第四，真实机械臂直接试错成本较高，容易造成碰撞、夹爪损坏或目标物体掉落，因此需要先在 ROS2 仿真环境中完成算法验证、轨迹规划验证和系统闭环测试。

因此，本项目需要视觉或 AI 系统重点解决以下问题：一是从摄像头采集的 RGB-D 图像或点云中感知场景结构，识别目标物体的位置、边界、深度和空间姿态；二是在多物体堆叠情况下生成多个候选抓取位姿，包括抓取点、抓取方向、夹爪姿态和开合宽度；三是对候选抓取进行评分和筛选，判断其可达性、稳定性和碰撞风险；四是将视觉坐标系中的抓取位姿转换到机械臂基坐标系，并交由 ROS2 / MoveIt 完成运动规划；五是在仿真环境中验证抓取策略，再逐步迁移到真实摄像头和真实机械臂系统中，实现从“视觉感知”到“抓取决策”再到“机械臂执行”的完整闭环。

项目最终目标是建立一套面向非结构化堆叠场景的机械臂智能抓取系统，使机械臂能够不依赖固定工装和人工示教，而是通过摄像头实时获取环境信息，利用 AI 模型估计抓取位姿，并通过 ROS2 完成运动规划与执行控制，从而提升机械臂在复杂场景下的自主抓取能力和部署适应性。

### 3.2 本期范围说明

本期重点完成 ROS2 仿真环境下的机械臂抓取闭环验证，核心范围包括：仿真场景搭建、相机数据获取、GraspNet 或同类抓取检测模型接入、候选抓取位姿生成、坐标转换、MoveIt 运动规划、夹爪动作控制、碰撞检测、抓取结果记录与可视化调试。

本期不重点展开真实工业产线部署、复杂上位机系统集成、长期稳定性压测和大规模模型泛化测试。真实相机、真实机械臂和真实夹爪的部署接口需要预留，但实际设备联调可作为后续阶段进行。

## 4. 输入与输出

### 4.1 输入

| 输入项 | 来源 | 格式/协议 | 说明 |
|---|---|---|---|
| RGB 图像 | 仿真相机或真实相机 | ROS2 Image Topic / PNG / JPG | 用于识别场景中目标物体的颜色、边界和表面特征。 |
| 深度图 | RGB-D 相机、深度相机或仿真平台 | ROS2 Image Topic / 16UC1 / 32FC1 | 用于获取物体表面到相机的深度距离。 |
| 点云数据 | RGB-D 相机或仿真平台 | ROS2 PointCloud2 / PCD / NPY | 用于 GraspNet 等模型生成三维抓取候选位姿。 |
| 相机内参 | 相机标定文件或仿真配置 | YAML / JSON | 包括焦距、主点、畸变参数等，用于深度图与点云计算。 |
| 相机外参/TF | 手眼标定结果或 ROS2 TF | TF Topic / YAML / JSON | 用于完成相机坐标系到机械臂基坐标系的转换。 |
| 机械臂模型 | URDF/Xacro 文件 | URDF / Xacro / SRDF | 描述机械臂关节、连杆、末端执行器和运动学约束。 |
| 夹爪参数 | 配置文件或仿真模型 | YAML / JSON | 包括夹爪最大开合宽度、最小开合宽度、抓取深度、夹持方向等。 |
| 目标物体模型 | 仿真平台或数据集 | STL / OBJ / USD / SDF | 用于构建多物体堆叠、遮挡和随机摆放场景。 |
| 业务任务信号 | 上位机、脚本或服务接口 | ROS2 Service / Action / JSON | 指定抓取任务类型，例如抓取单个物体、清空料框、抓取指定区域。 |
| 配置参数 | 配置文件或启动参数 | YAML / JSON / launch 参数 | 包括抓取阈值、候选数量、重试次数、安全距离和日志路径等。 |

### 4.2 输出

| 输出项 | 去向 | 格式/协议 | 说明 |
|---|---|---|---|
| 候选抓取结果 | 抓取决策模块、日志文件 | JSON / ROS2 Message | 输出多个候选抓取位姿，包括位置、姿态、评分、夹爪宽度和坐标系。 |
| 最终抓取位姿 | MoveIt 运动规划模块 | PoseStamped / Transform / JSON | 输出经过筛选后的最终抓取位姿，用于机械臂运动规划。 |
| 运动规划结果 | 机械臂控制模块、日志文件 | ROS2 Action Result / Trajectory | 输出规划是否成功、轨迹点、规划耗时和失败原因。 |
| 夹爪控制指令 | 夹爪控制接口 | ROS2 Topic / Service / Action | 控制夹爪张开、闭合、夹持和释放。 |
| 抓取执行状态 | 上位机、日志系统、可视化界面 | JSON / ROS2 Topic | 输出当前任务状态，例如识别中、规划中、抓取中、成功、失败。 |
| 结果图片 | 本地文件、调试界面 | PNG / JPG | 保存带有候选抓取姿态、最终抓取点和执行结果标注的图像。 |
| 点云与可视化数据 | RViz、仿真平台、本地文件 | PointCloud2 / PCD / MarkerArray | 用于显示目标点云、抓取方向、夹爪姿态和碰撞模型。 |
| 日志文件 | 本地或平台 | JSON / CSV / TXT | 保存输入数据、模型输出、筛选结果、执行状态、异常信息和评估指标。 |
| 评估统计结果 | 项目报告或测试系统 | CSV / Markdown / JSON | 输出抓取成功率、规划成功率、平均耗时、碰撞次数和失败类型统计。 |

## 5. 技术方案

### 5.1 总体处理流程

```text
任务触发
  -> 启动 ROS2 仿真场景
  -> 机械臂、夹爪、相机、目标物体加载
  -> 相机采集 RGB-D 图像/点云
  -> 图像与点云预处理
  -> GraspNet 或抓取检测模型生成候选抓取位姿
  -> 候选抓取评分、过滤和排序
  -> 相机坐标系位姿转换到机械臂基坐标系
  -> MoveIt 可达性判断与碰撞检测
  -> 生成预抓取点、抓取点、抬升点和放置点轨迹
  -> 夹爪执行张开、闭合、释放动作
  -> 输出抓取结果、日志和评估指标
```

### 5.2 系统模块划分

| 模块 | 主要功能 | 输入 | 输出 |
|---|---|---|---|
| 仿真场景模块 | 构建机械臂、夹爪、相机、目标物体和环境约束，支持多物体堆叠场景测试。 | URDF/Xacro、SDF/USD、物体模型、场景配置 | 可运行的仿真环境 |
| 相机采集模块 | 获取 RGB 图像、深度图和点云，提供给视觉/AI 模型。 | 仿真相机或真实相机数据流 | RGB-D 数据、点云数据 |
| 数据预处理模块 | 完成深度过滤、点云裁剪、坐标归一化、无效点剔除和 ROI 提取。 | RGB 图像、深度图、点云 | 预处理后的点云或模型输入张量 |
| 抓取检测模块 | 调用 GraspNet 或同类模型，生成多个候选抓取位姿。 | 预处理后的点云/RGB-D 数据 | 候选抓取位姿列表 |
| 抓取决策模块 | 根据评分、可达性、碰撞风险、夹爪约束筛选最终抓取位姿。 | 候选抓取位姿、机械臂状态、夹爪参数 | 最终抓取位姿 |
| 坐标转换模块 | 根据 TF 或手眼标定参数，将相机坐标系下的抓取位姿转换为机械臂基坐标系位姿。 | 候选位姿、TF 树、外参矩阵 | base_link 坐标系下的位姿 |
| 运动规划模块 | 通过 MoveIt 规划机械臂运动轨迹，完成可达性检查和碰撞检测。 | 抓取位姿、机械臂模型、场景碰撞体 | 规划轨迹和规划结果 |
| 执行控制模块 | 控制机械臂和夹爪完成预抓取、抓取、抬升、放置和复位动作。 | 轨迹、夹爪指令 | 执行状态和抓取结果 |
| 日志与评估模块 | 保存图像、点云、位姿、轨迹、状态和评估指标，支持问题追溯。 | 系统运行数据 | 日志文件、统计结果、可视化结果 |

### 5.3 技术路线

第一阶段完成仿真环境和基础闭环：搭建 ROS2 + MoveIt + Gazebo/Isaac Sim 环境，完成机械臂、夹爪、相机和多物体场景加载，验证机械臂基本运动规划和夹爪控制。

第二阶段完成视觉/AI 抓取检测：接入 GraspNet 或同类抓取检测模型，使用仿真 RGB-D 数据或点云生成候选抓取位姿，并统一输出格式。

第三阶段完成坐标转换和抓取执行：将相机坐标系下的抓取位姿转换到机械臂基坐标系，结合 MoveIt 完成预抓取点、抓取点、抬升点、放置点轨迹规划，并驱动夹爪完成抓取动作。

第四阶段完成多物体堆叠策略和评估：在随机堆叠场景中测试抓取成功率、规划成功率、碰撞次数、失败原因，优化候选位姿筛选和重试机制。

第五阶段预留真实设备迁移接口：整理真实相机、真实机械臂、真实夹爪的接入方式，准备手眼标定流程和真实部署安全约束。

### 5.4 异常处理策略

| 异常类型 | 处理策略 |
|---|---|
| 相机数据为空或深度无效 | 重新采集数据；若连续失败，终止任务并记录异常。 |
| 点云数量不足 | 放宽 ROI 或重新采集；仍不足时返回“感知失败”。 |
| 模型未输出有效抓取候选 | 调整抓取阈值或重新采样场景；仍失败时返回“无可抓取目标”。 |
| 坐标转换失败 | 检查 TF 是否存在、外参是否加载、frame_id 是否一致；失败时不执行机械臂动作。 |
| MoveIt 规划失败 | 切换下一个候选抓取位姿；候选全部失败时返回“规划失败”。 |
| 碰撞检测失败 | 剔除当前候选抓取位姿，选择下一个候选。 |
| 夹爪闭合失败 | 停止当前抓取流程，执行安全抬升或复位，并记录失败原因。 |
| 抓取后物体滑落 | 重新感知场景，进入重试逻辑。 |
| 仿真或节点异常退出 | 停止任务，保存已有日志，提示需要重新启动相关节点。 |

## 6. 算法或规则说明

| 模块 | 处理逻辑 | 关键参数 | 备注 |
|---|---|---|---|
| 图像采集 | 通过仿真相机或真实 RGB-D 相机采集 RGB 图像、深度图和点云，保证图像时间戳和坐标系一致。 | 分辨率、帧率、深度范围、camera_frame | 本期以仿真相机为主，真实相机接口预留。 |
| 点云预处理 | 对点云进行无效点剔除、深度范围过滤、ROI 裁剪、下采样和噪声过滤。 | 深度最小值、深度最大值、ROI 范围、体素大小 | 降低模型输入噪声，提高推理稳定性。 |
| 抓取检测 | 使用 GraspNet 或同类模型从点云中生成多个 6D 抓取候选，包括抓取中心、姿态、夹爪宽度和评分。 | 候选数量、评分阈值、夹爪宽度范围、模型权重路径 | 输出需保留原始模型评分，便于后续分析。 |
| 候选过滤 | 根据夹爪宽度、抓取高度、姿态角度、碰撞风险和机械臂可达性过滤不可执行候选。 | 最小/最大夹爪宽度、安全高度、姿态角限制、碰撞距离 | 过滤应发生在真实执行前，避免危险动作。 |
| 候选排序 | 综合模型评分、可达性、碰撞风险、抓取方向和机械臂运动代价进行排序。 | 模型评分权重、路径代价权重、碰撞惩罚权重 | 优先选择稳定且容易执行的抓取位姿。 |
| 坐标转换 | 根据 TF 树或手眼标定矩阵，将 camera_frame 下的抓取位姿转换到 base_link 坐标系。 | 外参矩阵、frame_id、时间戳、单位换算 | 坐标转换失败时不得执行运动规划。 |
| 预抓取点生成 | 沿抓取接近方向反向偏移一段安全距离，生成预抓取点。 | approach_distance、safe_height、抓取方向 | 用于避免夹爪直接碰撞目标或其他物体。 |
| 运动规划 | 使用 MoveIt 对预抓取点、抓取点、抬升点、放置点进行轨迹规划和碰撞检测。 | 规划器类型、规划时间、速度比例、加速度比例 | 规划失败时切换下一个候选位姿。 |
| 夹爪控制 | 抓取前张开夹爪，到达抓取位姿后闭合夹爪，抬升后保持夹持，到达放置点后释放。 | open_width、close_width、grasp_force、等待时间 | 仿真阶段可先使用开合宽度模拟，真实阶段需加入力控或电流反馈。 |
| 抓取结果判定 | 根据物体是否随夹爪抬升、目标位姿是否变化、夹爪闭合状态等判断抓取是否成功。 | 抬升高度、物体位移阈值、夹爪闭合阈值 | 仿真中可通过物体位姿变化判断，真实部署需结合视觉或夹爪反馈。 |
| 失败重试 | 抓取失败时重新感知场景或切换候选抓取，达到最大重试次数后返回失败。 | max_retry、候选数量、重新采集开关 | 防止系统无限循环。 |
| 日志追溯 | 保存输入数据、候选位姿、最终位姿、规划结果、执行状态和异常信息。 | 日志路径、保存格式、任务 ID | 用于定位失败原因和生成测试报告。 |

### 6.1 候选抓取筛选规则

候选抓取位姿需要同时满足以下条件后才能进入运动规划：

1. 抓取评分大于设定阈值。
2. 夹爪开合宽度处于设备允许范围内。
3. 抓取点位于有效工作空间内。
4. 抓取姿态未超出机械臂末端姿态限制。
5. 预抓取点、抓取点和抬升点均无明显碰撞风险。
6. MoveIt 能够规划出可执行轨迹。
7. 当前候选优先级高于其他可执行候选，或上一候选执行失败。

### 6.2 坐标转换规则

抓取检测模型输出的位姿默认位于相机坐标系下，执行前必须转换到机械臂基坐标系。转换链路如下：

```text
gripper_pose_camera
  -> camera_depth_optical_frame
  -> camera_link
  -> robot_base_link
  -> MoveIt planning frame
```

实际实现中需要保证 `frame_id`、时间戳、单位、旋转方向和四元数顺序一致。若 TF 查询失败、外参缺失或坐标系不一致，系统应终止当前抓取任务并输出明确错误信息。

## 7. 验收标准

| 序号 | 验收项 | 验收标准 | 验证方式 |
|---|---|---|---|
| 1 | 功能完整性 | 能够在 ROS2 仿真环境中完成“相机采集 -> 抓取检测 -> 坐标转换 -> MoveIt 规划 -> 夹爪抓取 -> 放置/复位”的完整闭环。 | 启动仿真任务，观察完整抓取流程是否能够执行并输出结果。 |
| 2 | 仿真环境可运行性 | 机械臂、夹爪、相机和目标物体能够正常加载，TF 树完整，关节状态和相机数据能够正常发布。 | 使用 RViz、ROS2 topic、TF 工具检查节点、话题和坐标系。 |
| 3 | 抓取位姿生成能力 | 系统能够基于 RGB-D 图像或点云输出多个候选 6D 抓取位姿，并包含评分、夹爪宽度和坐标系信息。 | 对固定测试场景运行抓取检测，检查候选抓取 JSON 或 ROS2 消息。 |
| 4 | 坐标转换正确性 | 相机坐标系下的抓取位姿能够正确转换到机械臂基坐标系，转换后位姿在 RViz 中与目标物体位置基本一致。 | 在 RViz 中叠加显示点云、抓取姿态和机械臂模型，检查空间关系。 |
| 5 | 运动规划成功率 | 在设定的测试场景中，候选抓取位姿经过筛选后能够完成 MoveIt 轨迹规划，基础场景规划成功率不低于 80%。 | 执行不少于 20 次仿真抓取测试，统计规划成功次数。 |
| 6 | 抓取成功率 | 在基础多物体堆叠仿真场景中，系统能够完成抓取、抬升和放置，初版抓取成功率不低于 60%。 | 执行不少于 20 次仿真抓取测试，统计成功抓取次数。 |
| 7 | 碰撞安全性 | 执行过程中不应出现机械臂明显穿模、夹爪撞击桌面、路径穿过障碍物等问题；碰撞候选应被过滤或重新规划。 | 开启仿真碰撞检测与 RViz 可视化，检查执行轨迹和碰撞日志。 |
| 8 | 多物体堆叠适应性 | 系统能够在多个物体随机摆放、部分遮挡或堆叠情况下生成可执行抓取候选，并至少完成一个目标物体抓取。 | 构建随机堆叠场景，执行多轮抓取测试并记录结果。 |
| 9 | 失败重试能力 | 当首个候选位姿规划失败或抓取失败时，系统能够切换下一个候选或重新感知，达到重试上限后返回明确失败原因。 | 人为设置不可达候选或低质量候选，检查重试流程和失败输出。 |
| 10 | 运行稳定性 | 连续运行多轮仿真抓取任务时，系统不应出现频繁节点崩溃、资源泄漏或任务卡死。 | 连续执行不少于 30 次仿真任务，检查节点状态和日志。 |
| 11 | 结果可追溯性 | 每次任务均应保存输入图像/点云、候选抓取位姿、最终执行位姿、运动规划结果、抓取状态和失败原因。 | 检查任务输出目录、JSON 日志、图片标注结果和统计文件。 |
| 12 | 文档完整性 | 提供环境安装、依赖配置、仿真启动、模型运行、任务执行、参数说明和测试方法文档。 | 按文档在新环境中复现基础仿真抓取流程。 |
